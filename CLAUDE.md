# Dotfiles

macOS-focused dotfiles for a dev environment.

## Repo Structure

Topic-based layout (Holman-style). Each top-level folder is a topic containing all related files.

```
git/           # gitconfig.symlink, git.src (aliases)
zsh/           # zshrc.mine.symlink, plugins, p10k theme, fns_widget.zsh
bash/          # bashrc.mine.symlink, bash.bash, init.bash, gitprompt submodule
dev/           # rails, heroku, pnpm, python, localstack, docker, sqitch, pryrc.symlink
homebrew/      # Brewfile, init.src (brew-wrap)
zoxide/        # init.zsh, init.bash
networking/    # networking.src (latmon, latofi, publicip)
utils/         # utils.src, fns.src (the fns picker)
bootstrap.sh   # Main install script
```

## File Extension Convention

- **`.symlink`** — linked to `~/.{basename}` by `bootstrap.sh`
- **`.src`** — sourced by both bash and zsh
- **`.zsh`** — sourced by zsh only
- **`.bash`** — sourced by bash only

New topic? Create a directory with a `.src` file — it's picked up automatically by both rc files.

## Bootstrap

```bash
./bootstrap.sh
```

Interactively:
1. Generates `git/gitconfig.local.symlink` from the `.example` template (prompts for name/email)
2. Asks which shell(s) to configure (bash / zsh / both)
3. Installs antidote via Homebrew if zsh is selected
4. Symlinks all `.symlink` files; handles conflicts (skip / overwrite / backup)
5. Appends `source ~/.zshrc.mine` (or `.bashrc.mine`) to the shell's main rc file

## Shell Manager

Zsh uses **antidote** (Homebrew). Plugins are in `zsh/zsh_plugins.txt.symlink`.

## Private Config

Both rc files source `~/.priv-dotfiles/` if it exists — use that for secrets/local overrides.

## Git Config

`git/gitconfig.symlink` has 30+ aliases. Local user config lives in `git/gitconfig.local.symlink` (git-ignored, generated by bootstrap).

## Documenting Functions for `fns`

`fns` (in `utils/fns.src`) is an interactive picker that surfaces documented shell functions and aliases. To make a function appear in the picker, place a comment in this exact format on the line immediately before the definition:

```sh
## @category Description with optional arg refs like arg{1} and arg{2}
my-function() { ... }
```

- **Category** — matches the topic name (e.g. `dev`, `git`, `rails`, `utils`, `networking`)
- **Arg refs** — use `{1}`, `{2}`, … to mark positional arguments in the description
- The `##` prefix (double hash) is required — single `#` comments are ignored by the parser

Examples from the codebase:

```sh
## @utils Extract various file types from a file{1}
extract() { ... }

## @rails Grep Rails routes matching pattern{1}
alias routes='bin/rails routes | grep'

## @dev List exported symbols from a dylib{1}, optionally filtered by prefix{2}
dylib-symbols() { ... }
```
