# Dotfiles

macOS-focused dotfiles for a Rails/Node/Go dev environment.

## Repo Structure

```
configs/       # Tool configs (git, pry, Brewfile)
shell/         # Shell config organized by topic
  *.src        # Top-level topic files (git, brew, rails, networking, etc.)
  bash/        # Bash-specific (aliases, gitprompt submodule)
  utils/       # General utilities (zoxide init)
  zsh/         # Zsh config, antidote plugin list, powerlevel10k theme, fns widget
bootstrap.sh   # Main install script
```

## Deployment Conventions

Two file suffixes drive the install:

- **`.symlink`** — linked to `~/.{basename}` by `bootstrap.sh` (e.g. `shell/zsh/zshrc.mine.symlink` → `~/.zshrc.mine`)
- **`.src`** — sourced at shell init time; both `bashrc.mine` and `zshrc.mine` glob `shell/*.src` and `shell/*/*.src`

New topic? Add a `shell/<topic>.src` file (or a subdirectory with a `.src` file) — it's picked up automatically.

## Bootstrap

```bash
./bootstrap.sh
```

Interactively:
1. Generates `configs/git/gitconfig.local.symlink` from the `.example` template (prompts for name/email)
2. Asks which shell(s) to configure (bash / zsh / both)
3. Installs antidote via Homebrew if zsh is selected
4. Symlinks all `.symlink` files; handles conflicts (skip / overwrite / backup)
5. Appends `source ~/.zshrc.mine` (or `.bashrc.mine`) to the shell's main rc file

## Shell Manager

Zsh uses **antidote** (Homebrew). Plugins are in `shell/zsh/zsh_plugins.txt.symlink`.

## Private Config

Both rc files source `~/.priv-dotfiles/` if it exists — use that for secrets/local overrides.

## Git Config

`configs/git/gitconfig.symlink` has 40+ aliases. Local user config lives in `configs/git/gitconfig.local.symlink` (git-ignored, generated by bootstrap).
